# CLI Client Review Remediation Plan

Branch: `feat/cli-client`  
PR: `#330` (`feat: add cli client foundation`)  
Source review date: 2026-05-01  
Approach: TDD for every code-facing change. Add failing tests first, implement the smallest green slice, then refactor while preserving architecture boundaries.

This plan converts the external deep review findings into an executable checklist. It is scoped to hardening the current CLI foundation before treating it as stable groundwork for later `run`, TUI, `serve`, `attach`, permissions, patch approval, WebUI, Telegram, and ACP work.

Legend: `[ ]` pending · `[~]` in progress · `[x]` done

## Goals

- [x] G.1 Stabilize public CLI JSON/NDJSON contracts before downstream clients depend on them.
- [x] G.2 Keep channel-neutral use cases and ports out of the CLI adapter module.
- [x] G.3 Make `golemcore-bot-app` the composition root for CLI runtime wiring.
- [x] G.4 Replace hand-written JSON with shared Jackson serialization.
- [x] G.5 Make command parsing deterministic: no silent typo acceptance, no discarded arguments, clear exit code `2` for invalid usage.
- [ ] G.6 Preserve current green CI while adding stronger contract, architecture, parser, and presenter tests.
